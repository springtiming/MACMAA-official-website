# 环境变量配置说明（Vercel / Supabase）

本文件是当前仓库的环境变量维护入口，说明哪些变量需要配置、应该配置到哪里，以及哪些历史变量名只保留兼容用途。

## 1. 配置原则

- 浏览器可读的变量统一使用 `NEXT_PUBLIC_` 前缀。
- 服务端密钥不要使用 `NEXT_PUBLIC_` 前缀，更不能把 `SUPABASE_SERVICE_ROLE_KEY` 放进公开变量。
- 代码中仍兼容一部分历史 `VITE_*` 变量名，但**新环境不要再使用这些旧名字**。
- 本地开发以仓库根目录 `.env.local` 为准，建议从 `env.template` 复制一份开始填写。
- Supabase Edge Functions 运行在 Supabase 侧，生产环境 Secrets 不从 Vercel 继承，必须单独配置。

## 2. 变量应该配置到哪里

### 本地开发

- 文件位置：仓库根目录 `.env.local`
- 用途：Next.js 页面、浏览器端 Supabase 调用、Vercel API Routes、本地邮件/支付/后台调试
- 起始模板：`env.template`

### Vercel 项目环境变量

- 配置位置：Vercel 项目 `Settings -> Environment Variables`
- 用途：生产 / Preview / Development 部署时的 Next.js 运行时与 API Routes
- 变更建议：修改后重新触发一次部署，确保新 Deployment 使用最新变量值

### Supabase Edge Function Secrets

- 配置位置：Supabase Dashboard 的 Edge Functions / Secrets
- 用途：`supabase/functions/*` 下所有部署到 Supabase 的函数
- 本地调试：如果要单独调试某个函数，建议显式传入根目录 `.env.local`

```bash
supabase functions serve news-media --env-file .env.local
```

- 生产设置可使用 Dashboard，或用 CLI 逐个写入：

```bash
supabase secrets set SUPABASE_URL=... --project-ref <project-ref>
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=... --project-ref <project-ref>
supabase secrets set JWT_SECRET=... --project-ref <project-ref>
supabase secrets set UNSPLASH_ACCESS_KEY=... --project-ref <project-ref>
```

## 3. Vercel / `.env.local` 变量清单

| 变量名                                   | 必需       | 用途                                          | 说明                                          |
| ---------------------------------------- | ---------- | --------------------------------------------- | --------------------------------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`               | 是         | 浏览器端 Supabase、Edge Function 调用基地址   | 公共值，可暴露给浏览器                        |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY`          | 是         | 浏览器端 Supabase、Edge Function 调用鉴权     | 公共 anon key，可暴露给浏览器                 |
| `NEXT_PUBLIC_SITE_URL`                   | 建议       | SEO、canonical、sitemap 绝对地址              | 生产环境建议明确设置                          |
| `NEXT_PUBLIC_ADMIN_API_BASE`             | 否         | 前端调用管理 API 基地址                       | 默认 `/api`，一般不需要改                     |
| `NEXT_PUBLIC_ENABLE_MOCK_ADMIN_ACCOUNTS` | 否         | 开发期 mock 管理员账号开关                    | 仅本地开发可用，Preview / Production 不要开启 |
| `SUPABASE_URL`                           | 是         | Next.js API Routes 的服务端 Supabase 客户端   | 通常与 `NEXT_PUBLIC_SUPABASE_URL` 相同        |
| `SUPABASE_SERVICE_ROLE_KEY`              | 是         | Next.js API Routes 的服务端 Supabase 管理权限 | 绝不能放到 `NEXT_PUBLIC_*`                    |
| `JWT_SECRET`                             | 是         | 自定义管理员 JWT 签发与校验                   | 必须与 Supabase Edge Functions 使用同一值     |
| `STRIPE_SECRET_KEY`                      | 视功能而定 | Stripe Checkout Server API                    | 只有启用在线支付时才需要                      |
| `STRIPE_WEBHOOK_SECRET`                  | 视功能而定 | 校验 `/api/payments/stripe-webhook` 回调签名  | 只有启用在线支付时才需要                      |
| `STRIPE_PAYMENT_CURRENCY`                | 否         | Stripe 默认货币                               | 默认 `aud`                                    |
| `RESEND_API_KEY`                         | 建议       | 邮件发送                                      | 未配置时流程不会报错，但邮件只会被跳过        |
| `RESEND_FROM_EMAIL`                      | 建议       | 邮件发件人                                    | 不配时回退到 `MACMAA <onboarding@resend.dev>` |
| `ADMIN_NOTIFICATION_EMAIL`               | 建议       | 管理通知收件邮箱                              | 会员申请、带备注的报名等通知会发到这里        |
| `TEST_EMAIL`                             | 否         | 邮件测试收件人                                | 仅测试与排查时用                              |
| `INITIAL_ADMIN_USERNAME`                 | 否         | 初始化管理员账号                              | 只在 `admin_accounts` 为空时生效              |
| `INITIAL_ADMIN_EMAIL`                    | 否         | 初始化管理员账号                              | 同上                                          |
| `INITIAL_ADMIN_PASSWORD`                 | 否         | 初始化管理员账号                              | 同上，建议首次初始化后从平台移除              |
| `SUPABASE_PROJECT_REF`                   | 否         | Supabase CLI / CI                             | 应用运行时不读取                              |
| `SUPABASE_ACCESS_TOKEN`                  | 否         | Supabase CLI / CI                             | 应用运行时不读取                              |

## 4. Supabase Edge Function Secrets 清单

以下变量需要配置到 Supabase，而不是只配在 Vercel：

| 变量名                      | 必需 | 使用位置                                                         | 说明                                                |
| --------------------------- | ---- | ---------------------------------------------------------------- | --------------------------------------------------- |
| `SUPABASE_URL`              | 是   | 几乎所有 `supabase/functions/*`                                  | Edge Functions 自己创建 Supabase 服务端客户端时使用 |
| `SUPABASE_SERVICE_ROLE_KEY` | 是   | 几乎所有 `supabase/functions/*`                                  | Edge Functions 访问数据库 / Storage 的管理权限      |
| `JWT_SECRET`                | 是   | `admin-auth` 以及所有通过 `_shared/auth.ts` 校验管理员身份的函数 | 必须与 Vercel API Routes 保持一致                   |
| `UNSPLASH_ACCESS_KEY`       | 按需 | `supabase/functions/unsplash`                                    | 只有后台封面图片搜索功能需要                        |

当前依赖这些 Secrets 的函数包括但不限于：

- `admin-auth`
- `activities`
- `events-admin`
- `events-registrations`
- `members`
- `news-admin`
- `news-drafts`
- `news-drafts-delete`
- `news-media`
- `news-publish`
- `payment-proofs`
- `review-logs`
- `volunteers`
- `unsplash`

## 5. 历史兼容变量（不要再新配）

代码仍兼容以下旧变量名，目的是平滑迁移旧环境；**新环境一律不要再使用它们**：

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_ADMIN_API_BASE`
- `VITE_ENABLE_MOCK_ADMIN_ACCOUNTS`
- `VITE_SUPABASE_SERVICE_ROLE_KEY`
- `ADMIN_EMAIL`
- `CONTACT_EMAIL`

标准做法始终优先：

- 公共变量使用 `NEXT_PUBLIC_*`
- 服务端变量使用无前缀的服务端专用名字

## 6. 维护建议

- 新增环境变量时，同时更新：
  - `env.template`
  - 本文件
  - 如有必要，`README.md` / `README_en.md`
- 修改 `JWT_SECRET` 后，Vercel API Routes 和 Supabase Edge Functions 必须一起更新，否则后台登录态会失效。
- 新的服务端密钥不要写进浏览器可见的代码路径；如果只是为了兼容旧逻辑才读取 `VITE_*`，后续应逐步清理。
- 交接账号时，真实密钥只保存在密码管理器、Vercel 控制台、Supabase Secrets、Stripe / Resend / Unsplash 控制台，不写入仓库文档。
