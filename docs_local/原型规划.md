## 原型目标
- 用最小可行界面串联核心业务流程：多语言入口、新闻浏览、活动报名（含支付方式选择）、会员申请、后台登录与内容管理门槛。
- 验证 Tailwind + Framer Motion + Icon CDN 的技术组合是否满足设计期望。
- 通过可点击演示收集团队反馈，决定正式开发的优先级与迭代节奏。

## 优先体验路径
1. **多语言入口**：展示自动语言检测与右上角切换，确保所有页面文本可双语切换。
2. **首页＋导航**：提供协会简介、CTA 入口（新闻/活动/会员/后台），强调导航一致性。
3. **新闻列表与详情**：基本列表卡片、时间戳、阅读更多按钮；详情页可返回。
4. **活动报名流程**：活动详情页 → 报名表 → 支付方式选择（按钮或模态） → 成功状态页。
5. **会员申请表**：收集核心字段（姓名、联系方式、背景、服务意向），提交后展示“已通知管理员”提示。
6. **后台登录与仪表盘雏形**：包括登录表单、活动/新闻管理列表占位、会员审核列表占位、账号管理入口。

## 原型阶段输出
- 可点击高保真页面（Figma 或静态 HTML 原型），覆盖上述路径。
- 文案与交互以中文为主、英文为辅，确保切换时布局不破坏。
- 每个流程附带注释说明真实系统将如何对接 API/邮件/支付。

## 视觉与动效规范
- 图标使用线条简洁、色彩温和的风格，避免过度商业或科技感。
- 动画保持轻量：按钮 hover、淡入淡出、图标轻微摆动即可，确保专业温暖。
- 色彩沿用主站暖米黄、浅绿、藏青等主色，不用冲突色。
- 建议优先使用 SVG 或 Lottie 动画，保证多设备清晰与流畅。

## 实施顺序
1. **定义信息架构**：梳理导航、路由、面包屑。
2. **确定视觉基调**：主色、辅色、字体、图标集，确保与 VMCA 品牌契合。
3. **制作公共组件**：导航、页脚、语言切换器、卡片、表单字段。
4. **搭建核心页面**：按“首页 → 新闻 → 活动 → 会员 → 后台”顺序构建，逐页验收。
5. **串联交互**：为按钮、切换器、表单添加过渡动画与状态反馈。
6. **标注业务对接点**：在原型中标明支付、通知、API 的触发位置。
7. **评审与调整**：收集利益相关人反馈，形成下一步开发的 backlog。

## 验收标准
- 全部关键路径可点击演示，语言切换正常。
- 每页明确指出后续真实数据/接口需求。
- 文档与原型链接归档至项目库，供开发与产品参考。
